# 個人密碼管理器 — 執行計畫

## 專案概述

打造一款以使用者為中心的個人密碼管理工具，強調自訂規則生成、便捷操作與查詢透明度，讓使用者對每一組密碼都有完整掌控權。

---

## 核心功能規劃

### 功能一：自訂規則密碼生成器

使用者在每次生成密碼前，需主動設定生成規則，避免被動接受預設值。

**規則選項包含：**
- 密碼長度（最短 8 位，最長 64 位）
- 字元類型（大寫字母、小寫字母、數字、特殊符號）
- 排除易混淆字元（如 `0`、`O`、`l`、`1`）
- 是否允許重複字元
- 自訂前綴 / 後綴（選填）

**流程：**
1. 使用者點擊「生成新密碼」
2. 彈出規則設定面板
3. 確認規則後，即時生成並預覽密碼
4. 可重新生成（保持相同規則）或修改規則後再生成

---

### 功能二：一鍵複製密碼

- 每筆密碼旁設置「複製」按鈕，點擊即複製至剪貼簿
- 複製後顯示短暫的成功提示（例如「已複製！」持續 2 秒）
- 基於安全考量，剪貼簿內容於 30 秒後自動清除，並提示使用者

---

### 功能三：網站密碼查詢

集中管理各網站對應的密碼紀錄，支援快速查詢。

**每筆紀錄包含：**
- 網站名稱 / 網址
- 帳號（Email 或使用者名稱）
- 密碼（預設遮蔽，可點擊顯示）
- 建立日期 / 最後更新日期
- 備註（選填）

**查詢功能：**
- 依網站名稱或帳號關鍵字搜尋
- 依建立時間或字母排序
- 標記「常用」網站，置頂顯示

---

## 技術架構

| 層級 | 技術選型 |
|------|----------|
| 前端介面 | React + Tailwind CSS |
| 本地儲存 | IndexedDB（加密儲存） |
| 加密方案 | AES-256，以主密碼派生金鑰（PBKDF2） |
| 剪貼簿操作 | Clipboard API |
| 部署方式 | 純前端靜態應用，可本機運行或部署至靜態主機 |

---

## 安全設計原則

1. **主密碼不儲存**：僅用於派生加密金鑰，原始密碼永不寫入磁碟
2. **全程本地加密**：密碼資料加密後才寫入 IndexedDB，明文不出現在儲存層
3. **自動鎖定**：閒置超過 5 分鐘（可設定）後自動鎖定，需重新輸入主密碼
4. **無網路傳輸**：純離線應用，資料不上傳任何伺服器
5. **匯出加密備份**：支援以加密 JSON 格式匯出，方便備份與遷移

## 成功指標

- 生成一組符合規則的密碼 **< 5 秒**
- 查詢任意已儲存網站密碼 **< 3 秒**
- 所有敏感資料均以加密形式儲存，明文不可在儲存層讀取
